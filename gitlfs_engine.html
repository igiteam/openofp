🎮 Game Engine Git LFS Manager

What These Scripts Do

These cross-platform scripts (Bash for Linux/macOS and PowerShell for Windows) automatically configure Git LFS (Large File Storage) for game development projects across 8 different game engines. They ensure large binary assets are properly tracked and managed in version control.

🏗️ System Architecture

[Game Project] → [Git LFS Manager] → [Git LFS Tracking]
     ↓                    ↓              ↓
  Engine-specific    Auto-detection   Large file
  asset files        & pattern        optimization
                     application

# engine_gitlfs.sh
ENGINE="${1:-}"

# Interactive selection
if [[ -z "$ENGINE" ]]; then

    # ... engine list display

🪟 PowerShell Script (Windows)

#!/usr/bin/env pwsh
# engine_gitlfs.ps1
param(
    [string]$Engine = "",
    [string]$DoPush = ""
)

# Interactive selection
if ([string]::IsNullOrEmpty($Engine)) {
    Write-Host "Please select a game engine:"
    # ... engine list display
    $choice = Read-Host "Enter number"
    $Engine = $SupportedEngines[[int]$choice - 1]
}

# Apply LFS tracking
git lfs track $pattern

Features:

  • PowerShell cmdlets and objects
  • Windows-native file operations
  • Object-oriented approach
  • Rich error handling with try/catch

🛠️ Installation & Setup

Step 1: Dependency Check

Automatically verifies Git and Git LFS installation, with installation prompts if missing.

# Bash
if ! command -v git-lfs >/dev/null 2>&1; then
    echo "📦 Installing Git LFS..."
    sudo apt-get install -y git-lfs
fi

# PowerShell
if (-not (Get-Command git-lfs -ErrorAction SilentlyContinue)) {
    throw "Git LFS not found. Please install from: https://git-lfs.github.com/"
}

Step 2: Repository Validation

Ensures script runs within a valid Git repository before making changes.

# Bash
if ! git rev-parse --git-dir > /dev/null 2>&1; then
    echo "❌ Error: This is not a git repository"
    exit 1
fi

# PowerShell
$gitCheck = git rev-parse --is-inside-work-tree 2>&1
if ($LASTEXITCODE -ne 0) {
    throw "Not a git repository."
}

Step 3: Interactive Engine Selection

Presents numbered list of supported engines with input validation.

Step 4: Large File Scanning

Scans for files larger than 10MB to identify candidates for LFS tracking.

# Bash
find . -type f -size +10M -exec ls -lh {} \;

# PowerShell
$largeFiles = Get-ChildItem -Recurse -File | Where-Object { $_.Length -gt 10MB }

Step 5: LFS Pattern Application

Applies engine-specific file patterns to Git LFS tracking.

# Both scripts use same pattern approach
git lfs track "*.upk"
git lfs track "*.umap"
git lfs track "*.uasset"
